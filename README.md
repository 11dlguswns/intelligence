# Claude Intelligence Monitor

당신의 **Claude 플랜(구독) 계정**으로, 각 Claude 모델이 **자기 최고 실력 대비** 지금 얼마나 떨어졌는지
— 즉 "쓰다 보니 멍청해졌나"를 시간단위로 추적하는 도구입니다.

> 🔗 라이브: https://11dlguswns.github.io/intelligence/

---

## 무엇을, 왜 이렇게 재나

핵심 원칙: **매 측정마다 문제를 새로 생성**합니다. 숫자·조건이 매번 달라지는 12문제(6차원)를 풀게 하고,
**정답도 코드가 직접 계산**해 정확히 채점합니다 — LLM 심사위원 없음.

왜 고정된 유명 퍼즐을 안 쓰나:

- **오염(암기) 차단**: 12공 저울·몬티홀 같은 고전은 훈련 데이터에 있어 *외운 답*을 떠올릴 뿐입니다.
  멍청해진 모델도 외운 답은 맞혀 — 저하가 가려집니다. 새 인스턴스는 *실제 추론*을 강제하므로
  약해진 모델이 눈에 띄게 틀립니다. (GSM-Symbolic 연구: 숫자만 바꿔도 '풀던' 벤치마크 점수가 떨어짐.)
- **노이즈↓**: 정답을 코드가 계산해 정확 채점 → 점수 변화 = 실제 변화.
- **비용 절반**: 답변 호출만, 심사위원 호출 0.

6개 차원 × 각 2문제: **추론**(기사와 건달·줄 세우기) · **수학**(공장 다단계·연립) · **확률**(비복원 추출·기댓값) ·
**공간**(색칠 정육면체·격자 경로) · **조합**(포함배제·동전 교환) · **절차**(콜라츠 반복·반복 모듈러).

각 모델의 **최고점(역대 최고 = 진짜 실력)** 기준으로 현재가 얼마나 떨어졌는지 봅니다(8점↓ 🟡, 18점↓ 🔴).

> 정직한 한계: **건강한 프런티어 모델(Opus 등)은 만점에 가깝습니다** — 기계적으로 풀리는 문제는 단계적 사고로
> 천장에 닿기 때문입니다. 결함이 아니라 사실이고, 이 도구의 목적(만점에서 *떨어지는 순간* 포착)엔 오히려 맞습니다.
> 긴 계산(콜라츠·모듈러)에서 강한 모델도 가끔 실수해 변별이 생기고, 작은 모델일수록 더 출렁입니다.

## 아키텍처 (수집 ↔ 표시 분리)

구독 OAuth는 공개 서버에 못 올리므로, **로컬 러너가 측정**하고 **결과 JSON만 정적 사이트**에 올립니다.

```
 [당신 PC] 러너 (claude 로그인)                    [GitHub Pages] 정적 대시보드
   문제 생성 → 모델 답변 → 정답 자동 채점 → JSON ── git push ──▶   최고점 대비 현재 + 차원별 막대 + 추이
```

- 러너 `packages/runner` (의존성 0): `problems.mjs`가 문제 생성+채점, `claude -p` 헤드리스(도구 OFF·effort 고정).
- 웹 `packages/web` (Vite+React): 모델별 종합 점수·6차원 가로 막대·최고점 대비·스파크라인, LIVE·25초 자동 새로고침.

## 빠른 시작

```bash
npm install
npm run bench -- --models opus,sonnet,haiku   # 1회 측정 (반복 시 최고점/기준 형성)
npm run web:dev                                # http://localhost:5173/intelligence/
```

## 시간단위 자동 측정 (실시간 추적)

```powershell
# 새 PowerShell 창에서 (프로젝트 폴더)
./packages/runner/schedule.ps1 -IntervalMinutes 120    # 2시간마다 측정→push→자동 배포 (기본)
#   -IntervalMinutes 240 / -Models "opus" 등으로 조절. 0 = 매일 1회(-Time)
#   절전 깨우기·배터리 허용 포함. 단, 완전 종료/최대절전이면 못 깨움(PC가 켜져 있어야 함).
# 제거: Unregister-ScheduledTask -TaskName "ClaudeIntelligenceMonitor" -Confirm:$false
```

> ⚠️ 비용: 매 측정 = N모델 × 12문제 (심사위원 없음 → 이전의 절반). 레이트리밋 시 간격↑/모델↓.

## 배포 (GitHub Pages)

repo push → Settings → Pages → Source = GitHub Actions. `.github/workflows/deploy.yml`가 빌드·배포(base 경로 자동).

## 한계 / 주의

- **천장**: 건강한 상위 모델은 ~100점입니다. 변별·저하는 주로 작은 모델과 긴 계산 문제에서, 그리고
  점수가 *최고점에서 떨어질 때* 나타납니다. 만점이 평탄한 건 "지금 멀쩡함"이라는 정직한 신호입니다.
- **인스턴스 변동**: 매번 다른 문제라 난이도가 조금씩 달라 한 번의 점수는 출렁입니다. 12문제 평균·최고점
  기준·시간별 추이로 완화하되, 단일 하락보다 *지속 하락*이 진짜 저하입니다.
- 정답은 코드가 계산하므로 채점 자체는 정확합니다(LLM 심사위원 편향 없음). 비공식 측정입니다.
- 문제 생성기·effort를 바꾸면 새 시리즈로 보세요.
