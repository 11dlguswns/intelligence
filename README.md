# Claude Intelligence Monitor

당신의 **Claude 플랜(구독) 계정**으로, 각 Claude 모델이 **자기 최고 실력 대비** 지금 얼마나 떨어졌는지
— 즉 "쓰다 보니 멍청해졌나"를 시간단위로 추적하는 도구입니다.

> 🔗 라이브: https://11dlguswns.github.io/intelligence/

---

## 무엇을, 왜 이렇게 재나

두 신호를 **하이브리드**로 봅니다.

**① 🛡 객관 건강도 — 저하 감지(정직한 핵심)**
매 측정마다 **문제를 새로 생성**하고(숫자·조건이 매번 달라짐) **정답도 코드가 직접 계산**해 정확 채점합니다.
6개 차원을 쉬움·중간·어려움 3단계로 풀려 난이도 가중 통과율을 냅니다.
- **오염(암기) 차단**: 고전 퍼즐은 *외운 답*을 떠올릴 뿐이라 멍청해져도 가려집니다. 새 인스턴스는 *실제 추론*을
  강제 → 약해지면 못 풉니다. (GSM-Symbolic: 숫자만 바꿔도 '풀던' 점수가 떨어짐.)
- 코드 채점이라 편향 0. 평소 풀던 걸 못 풀면 떨어지는 **가장 믿을 수 있는 저하 신호**.

**② 추론 품질 — 화면의 큰 숫자·막대(변별)**
건강한 Claude들은 정답 맞히기로는 다 천장(난이도를 올려도 100)이라 서로 구분이 안 됩니다. 그래서 어려운 문제
풀이를 **독립된 Opus 심사위원이 0~100 채점**해 *추론의 질*로 갈라 보여줍니다.
- 코드가 계산한 **정답을 레퍼런스로** 제공해 채점. 정답을 맞히면 **88점 이상으로 바닥 고정** → 심사위원의
  장황함 편향·노이즈가 *맞은 답을 깎지 못하게* 막았습니다(이게 없으면 간결한 Opus가 장황한 Haiku보다 낮게 나옴).

6개 차원: **추론**(기사와 건달) · **수학**(연산 사슬) · **확률**(공 뽑기) · **공간**(격자 경로) · **조합**(동전 교환) · **절차**(반복 모듈러).
각 모델의 **최고점 대비** 품질 또는 건강도가 떨어지면 🟡주의·🔴멍청해짐.

> 정직한 한계: 건강한 상위 모델은 품질도 ~90~100에 몰립니다(정말로 다 잘함). 모델 간 변별은 미세하고, 진짜
> 가치는 *시간에 따라 떨어지는지*입니다 — 객관 건강도가 그 저하를 암기 불가·코드 채점으로 가장 믿을 수 있게 잡습니다.

## 아키텍처 (수집 ↔ 표시 분리)

구독 OAuth는 공개 서버에 못 올리므로, **로컬 러너가 측정**하고 **결과 JSON만 정적 사이트**에 올립니다.

```
 [당신 PC] 러너 (claude 로그인)                       [GitHub Pages] 정적 대시보드
   문제생성 → 답변 → ①코드 채점(건강도)+②Opus 심사(품질) → JSON ──▶  최고점 대비 + 차원 막대 + 추이
```

- 러너 `packages/runner` (의존성 0): `problems.mjs`(문제 생성+사다리 채점) + `quality.mjs`(Opus 심사), `claude -p` 헤드리스.
- 웹 `packages/web` (Vite+React): 모델별 품질 점수·🛡객관 건강도·6차원 막대·최고점 대비·스파크라인, LIVE·25초 자동 새로고침.

## 빠른 시작

```bash
npm install
npm run bench -- --models opus,sonnet,haiku   # 1회 측정 (반복 시 최고점/기준 형성)
npm run web:dev                                # http://localhost:5173/intelligence/
```

## 시간단위 자동 측정 (실시간 추적)

```powershell
# 새 PowerShell 창에서 (프로젝트 폴더)
./packages/runner/schedule.ps1 -IntervalMinutes 120    # 2시간마다 측정→push→자동 배포 (기본)
#   -IntervalMinutes 240 / -Models "opus" 등으로 조절. 0 = 매일 1회(-Time)
#   절전 깨우기·배터리 허용 포함. 단, 완전 종료/최대절전이면 못 깨움(PC가 켜져 있어야 함).
# 제거: Unregister-ScheduledTask -TaskName "ClaudeIntelligenceMonitor" -Confirm:$false
```

> ⚠️ 비용: 매 측정 = 모델당 약 24호출(사다리 18 + 심사 6). 심사위원이 Opus라 부담이 있습니다 — 레이트리밋 시 `-IntervalMinutes`↑ 또는 `-Models`↓.

## 배포 (GitHub Pages)

repo push → Settings → Pages → Source = GitHub Actions. `.github/workflows/deploy.yml`가 빌드·배포(base 경로 자동).

## 한계 / 주의

- **천장**: 건강한 상위 모델은 객관·품질 둘 다 ~90~100에 몰립니다(정말 다 잘함). 모델 간 변별은 미세하니,
  진짜로 봐야 할 건 *자기 최고점에서 떨어지는지*입니다.
- **심사위원 한계**: 품질 점수는 LLM 채점이라 노이즈·편향이 있습니다. 코드가 아는 정답으로 *맞은 답은 88점 바닥
  고정*해 가장 큰 왜곡(장황함 편향)을 막았지만, 미세한 출렁임은 남습니다. **저하 판단의 핵심은 객관 건강도**입니다.
- **인스턴스 변동**: 매번 다른 문제라 한 번의 점수는 출렁입니다. 최고점 기준·시간별 추이로 완화하되,
  단일 하락보다 *지속 하락*이 진짜 저하입니다.
- 문제 생성기·심사 기준·effort를 바꾸면 새 시리즈로 보세요. 비공식 측정입니다.
